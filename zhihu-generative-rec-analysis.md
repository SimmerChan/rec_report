三、各互联网厂商技术演进方向
================================================================================

【Meta】2024年2月 — 开创者，工业界标杆
  技术: GR (Generative Recommendation) + HSTU (High-Speed Transformer for User)
  核心: 统一序列建模，验证Scaling Law存在，主业务端+12.4%效果提升
  最新: ULTRA-HSTU (2026.2) — 18层16k序列，5x训练+21x推理加速
  知乎评价: "Meta GR推出以来，生成式推荐就是推荐系统领域最火的话题"

【字节/抖音】2025年 — 选择判别式Scaling路线
  技术: HLLM → OneTrans → RankMixer → TokenMixer-Large (2026.2)
  核心: 判别式Scaling Law，7B/15B参数，Token Mixing，MoE架构
  最新: UG-Separation (2026.2) — 推理延迟-20%，W8A16量化
  知乎评价: "字节全程没有明显押注生成式，选择判别式路线"
  代表观点: "抖音反其道而行之，在判别式精排模型上实现Scaling Law"

【快手】2025年2月 → 2026年4月 — 先生成后判别
  技术: OneRec (生成式) → UniMixer (判别式统一架构)

  创新: Encoder-Decoder → Lazy Decoder-Only，RL偏好对齐
  知乎评价: "快手onerec是业界最值得讨论的推荐系统框架"
  转向原因: "快手转而发UniMixer，专注判别式backbone的scaling路线"

【美团】2025年 — 多篇积累，工业落地

  核心: 保留DLRM交叉特征，Group-LN，动态掩码，广告端到端
  创新: 曝光日志做负样本，30分钟窗口聚合，混合序列

五、技术路线争议焦点
================================================================================

【路线一: 生成式GR路线】
  代表厂商: Meta GR, 快手OneRec, 美团MTGR
  技术主张: 端到端生成，统一建模，Next Token Prediction
  核心优势:
  - 消除级联误差传播
  - 超长序列全局优化
  - 统一用户意图理解
  主要问题:
  - 延迟高 (P99延迟约束)
  - 成本高 (算力消耗大)
  - 可控性差 (输出不稳定)
  - 广告落地难 (出价/计费逻辑)

【路线二: 判别式Scaling路线】
  代表厂商: 字节RankMixer/TokenMixer-Large, 快手UniMixer
  技术主张: 保留判别式架构，享受Scaling Law红利
  核心优势:
  - 低延迟 (矩阵并行计算)
  - 高可控 (pointwise预估)
  - 工程友好 (复用现有架构)
  - 效果稳定 (可解释性强)
  代表观点 (九河之间): "Meta/快手赌范式革命，抖音选择极致进化，
                       将判别模型能力scaling到新高度"

【路线三: 混合架构路线】
  代表厂商: 美团MTGR
  技术主张: 兼容传统DLRM特征体系，逐步演进
  核心优势:
  - 信息损失小
  - 重构成本低
  - 渐进式升级
  实践方案: Group-wise Processing, 动态掩码, 混合序列

六、核心技术挑战 (按讨论频率排序)
================================================================================
1. 算力成本高 (53次提及)
   - 传统DLRM训练MFU仅4.6%，推理MFU 11.2%
   - LLM训练MFU可达40-50%
   - 解决思路: M-FALCON并行化, BF16/FP8混合精度, INT4量化

2. 延迟问题 (32次提及)
   - 严格P99延迟约束 (≤200ms)
   - 千级候选处理瓶颈
   - 解决思路: KV缓存共享, 分组查询注意力(GQA), 算子融合

3. 可控性问题 (25次提及)
   - 调参效果不稳定
   - 输出难以精确控制
   - 解决思路: 强化学习偏好对齐, 奖励塑形

4. 冷启动问题 (21次提及)
   - 新用户/物品无历史行为
   - 解决思路: 语义ID层次量化, 多模态特征融合

5. 幻觉问题 (11次提及)
   - 生成不存在商品的ID
   - 解决思路: Semantic ID校验, Rerank阶段兜底

6. Ground Truth缺失 (6次提及)
   - 无明确Next Item标签
   - 解决思路: 曝光日志做负样本, 窗口聚合策略

7. 广告落地难 (3次提及)
   - 出价、计费逻辑难以融合
   - 解决思路: 小型判别式模型二次预估

七、关键结论
================================================================================

1. 范式转变: 不是"是否革命"而是"如何融合"
   - 短期(1-2年): 作为新组件嵌入现有系统
   - 中期(3-5年): 部分场景替代传统级联
   - 长期: 端到端统一是方向但道路漫长

2. 技术收敛: 两条路都在探索，判别式更务实
   - 生成式: OneRec/MTGR验证可行性，但未大规模替换
   - 判别式: RankMixer/UniMixer验证Scaling Law，更务实

3. 落地挑战: 成本+延迟+可控性三座大山
   - 不是技术不可行，而是ROI算不过来
   - 需要业务场景足够简单、收益足够大

4. 业界态度: 从狂热到理性
   - 2024-2025上半年: 争相Follow Meta GR
   - 2025下半年-2026: 观望等待，回归务实
   - 2026年趋势: 判别式Scaling路线更受青睐

5. 未来展望: 下一步的三个方向
   - 推理效率优化 (ULTRA-HSTU, xGR)
   - 多模态融合 (VLM时代的数据基础设施)
   - 统一范式探索 (UniMixer, UG-Separation)
